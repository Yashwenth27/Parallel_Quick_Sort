Parallel Quicksort with CUDA

Using a Global Stack for Dynamic Work Distribution

The Problem: Why Parallel Sorting?

  • Fundamental Task: Sorting is a core computing operation.
  • The Bottleneck: For massive datasets, single-core CPU sorting is too slow.
  • The Goal: Leverage thousands of GPU cores to sort data dramatically faster.
  • The Challenge: How do we efficiently divide the work and keep all cores busy?

The Algorithm: A "To-Do List" Approach

We use a "divide and conquer" strategy with a shared to-do list, called a **stack**.

  1. Start with one task: "Sort the whole array".
  2. A thread grabs a task and **partitions** the array.
  3. It adds the LARGER new part back to the to-do list.
  4. It immediately works on the SMALLER part.

This "work-stealing" model keeps all threads busy automatically!

  • #define: Defines constants. `INSERTION_SORT_THRESHOLD` is a key tuning parameter for performance.
  • struct SubArray: A simple data structure to hold a sorting task (the start and end indices of a subarray).

Part 2: Device Helper Functions

The `__device__` keyword means these functions are compiled to run on the GPU. They are the tools each worker thread uses.

  • swap: A basic utility to swap two numbers.
  • insertion_sort: A simple sort that is very fast for small arrays. Used to finish off tiny tasks.
  • partition: The core Quicksort logic. It rearranges a subarray around a pivot element.
__global__ void parallel_quicksort_kernel(...) { while (*sorted_count < n) { int current_stack_pos = atomicSub(stack_top, 1); if (current_stack_pos > 0) { SubArray task = stack[current_stack_pos - 1]; int low = task.low; int high = task.high; while (low < high) { if ((high - low + 1) < INSERTION_SORT_THRESHOLD) { insertion_sort(arr, low, high); atomicAdd(sorted_count, high - low + 1); break; } int pivot_index = partition(arr, low, high); atomicAdd(sorted_count, 1); if ((pivot_index - 1 - low) > (high - (pivot_index + 1))) { // Push left part, process right atomicAdd(stack_top, 1); low = pivot_index + 1; } else { // Push right part, process left atomicAdd(stack_top, 1); high = pivot_index - 1; } } if(low == high) { atomicAdd(sorted_count, 1); } } } }

Part 3: The Kernel (The GPU's Brain)

The `__global__` function is the main entry point for the GPU threads. This is where the parallel logic lives.

  • The Main Loop: Threads continuously work until all elements are sorted.
  • `atomicSub`: A thread safely takes a task from the global stack. This is the "work-stealing" step.
  • Process Task: The thread partitions its subarray.
  • `atomicAdd`: The thread pushes the larger of the two new subarrays back onto the stack for another thread to find. This is the "delegate" step.
int main() { const int N = 1 << 10; std::vector<int> h_arr(N); // ... initialize data ... // Sort on CPU for validation std::sort(h_cpu_sorted.begin(), h_cpu_sorted.end()); // Allocate memory on GPU int* d_arr; cudaMalloc((void**)&d_arr, N * sizeof(int)); // ... malloc stack, etc. ... // Copy data from Host to Device cudaMemcpy(d_arr, h_arr.data(), ..., cudaMemcpyHostToDevice); // Launch the Kernel on the GPU parallel_quicksort_kernel<<<blocks, threads>>>(...); // Copy sorted data from Device to Host cudaMemcpy(h_gpu_result.data(), d_arr, ..., cudaMemcpyDeviceToHost); // Validate and clean up cudaFree(d_arr); return 0; }

Part 4: The Host (The CPU's Control Room)

The `main` function runs on the CPU and manages the entire process from start to finish.

  1. Setup: Creates the array and a sorted copy for validation.
  2. `cudaMalloc`: Reserves memory on the GPU's RAM.
  3. `cudaMemcpy`: Copies the unsorted array to the GPU.
  4. Kernel Launch `<<<...>>>`: This is the command that starts the thousands of threads running the kernel code on the GPU.
  5. Retrieve Result: Copies the sorted array back from the GPU.
  6. Cleanup: Frees the GPU memory.

Performance & Conclusion

Results

For small arrays (N < 100,000), CPU can be faster due to data transfer overhead. For large arrays, the GPU's parallelism provides a massive speedup.

Conclusion

The global stack model is a powerful technique for **dynamic load balancing** in parallel algorithms. CUDA unlocks the massive power of GPUs for general-purpose tasks like sorting, making large-scale data processing feasible.
